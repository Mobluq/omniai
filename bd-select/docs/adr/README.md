# Architecture Decision Records

Use ADRs for decisions that shape future development.

## When to Add an ADR

Add an ADR when a change affects:

- architecture
- database ownership or migration strategy
- security posture
- deployment or CI/CD
- major dependencies
- public API contracts
- long-term product constraints

## Format

Use a numbered file:

```text
0005-short-decision-title.md
```

Each ADR should include status, context, decision, and consequences.
