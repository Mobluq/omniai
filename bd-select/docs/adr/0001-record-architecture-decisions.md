# 0001. Record Architecture Decisions

Status: Accepted

## Context

The project is expected to be long-lived and team-owned. Important choices need to survive handoff, onboarding, and product pivots.

## Decision

Architecture decisions will be recorded as ADRs under `docs/adr`.

## Consequences

- Future engineers can understand why core choices were made.
- Large changes require a written decision before implementation.
- Some documentation overhead is accepted to reduce long-term ambiguity.
