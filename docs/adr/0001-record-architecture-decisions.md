# 0001. Record Architecture Decisions

Status: Accepted

## Context

This project is expected to be long-lived and handled by a team. Important choices need to survive handoff, onboarding, and future product pivots.

## Decision

Architecture decisions will be recorded as ADRs under `docs/adr`. Decisions should be concise, dated through git history, and reviewed like code.

## Consequences

- Future engineers can understand why core choices were made.
- Large changes require a written decision before implementation.
- Some documentation overhead is accepted to reduce long-term ambiguity.
