# Documentation Index

This directory contains the project-neutral foundation docs that let a team continue development safely before the final product scope is known.

## Foundation

- `foundation/PRODUCT_BRIEF_TEMPLATE.md`: intake template for the future product brief.
- `foundation/ENGINEERING_BASELINE.md`: code organization, ownership, testing, and delivery rules.
- `foundation/NAMING_CONVENTIONS.md`: naming rules for modules, routes, schema, branches, env vars, and docs.
- `foundation/DATABASE_PLAYBOOK.md`: Prisma/PostgreSQL modeling, migration, and tenant data standards.
- `foundation/SECURITY_BASELINE.md`: auth, authorization, secrets, rate limits, audit logs, and AI data handling.
- `foundation/CI_CD_PLAYBOOK.md`: validation, deployment, release, rollback, and branch protection expectations.
- `foundation/HANDOFF_CHECKLIST.md`: onboarding checklist for new engineers and future feature work.

## Decisions

- `adr/README.md`: how to record architectural decisions.
- `adr/0001-record-architecture-decisions.md`: ADR process.
- `adr/0002-use-a-modular-nextjs-saas-baseline.md`: initial architecture decision.
- `adr/0003-prioritize-tenant-isolation-and-auditability.md`: security and data ownership decision.
- `adr/0004-manage-database-change-through-prisma-migrations.md`: database migration decision.
